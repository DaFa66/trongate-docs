_attempt_get_valid_token()

function _attempt_get_valid_token(int|array|null $user_levels = null): string|false

Description

Attempt to validate and return a token based on optional user level(s) condition. This method checks for a token in both the user's cookie and session. If no tokens are found, it returns false.

Parameters

The $user_levels parameter specifies the user levels to filter tokens. It is an optional argument that can accept different types of inputs:

  • Integer: If an integer is passed, the method will attempt to match a token against records from the Trongate User Levels table ('trongate_user_levels') where the 'id' from the 'trongate_user_levels' table matches the integer value provided. In this scenario, a token will only be returned successfully if it can be matched against the required Trongate User Level ID.
  • Array: If an array of integers is passed, each array item represents a user level ID for which a token can be matched. The method will allow access for more than one user level type.
  • Null: If no argument is passed into '_attempt_get_valid_token()', any valid Trongate token (as confirmed by querying the 'trongate_tokens' table) will be returned, regardless of the corresponding user level.

Return Value

Type Description
string|false The valid token if found, or false if none is found.

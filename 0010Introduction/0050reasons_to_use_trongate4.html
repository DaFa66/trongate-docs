Reasons To Use Trongate

Reason #4 : Productivity

Trongate can make you a productivity powerhouse. Getting up and running requires no Composer, no third-party libraries and no use of the command line.

From a syntax perspective, Trongate uses pure PHP. That means, it's simple, easy to learn and most importantly - it lets developers get up and running quickly.

For the purposes of comparison, let's compare some typical Trongate syntax with the the syntax of some other popular PHP frameworks.

Fetching A Single Record Of A Given 'id'

Symfony

$entityManager = $this->getDoctrine()->getManager();

$record_obj = $entityManager->getRepository(Entity::class)->find($id);

CodeIgniter

$this->db->where('id', $id);

$query = $this->db->get('table_name');

$record_obj = $query->row();

CakePHP

$record_obj = $this->ModelName->find('first', array(

    'conditions' => array('ModelName.id' => $id)

));

Trongate

$record_obj = $this->model->get('id');

Reading a Value from the Third URL Segment

Laravel

use Illuminate\Http\Request;

public function index(Request $request)
{
    $id = $request->segment(3);
}

Symfony

use Symfony\Component\HttpFoundation\Request;

public function index(Request $request)
{
    $id = $request->attributes->get('id');
}

Yii

use yii\web\Request;

public function actionIndex()
{
    $request = new Request();
    $id = $request->getQueryParam('id');
}

Trongate

$id = segment(3);

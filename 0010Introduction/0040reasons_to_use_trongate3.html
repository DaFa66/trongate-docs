Reasons to Use Trongate

Reason #3: No Third-Party Dependencies

While integrating third-party code can offer benefits, relying on third-party code can introduce challenges such as:

  • Being forced to adhere to someone else's rewrite schedule.
  • Inheriting security vulnerabilities of external libraries.
  • Aligning with the vision of external library maintainers.

We Did The Unthinkable!

Instead of cozying up to third-party providers, the makers of Trongate choose a different path. We wrote our own libraries! This includes but is not limited to:

  • A complete eco-system of JavaScript components
  • A minimalist CSS framework
  • A code-sharing ecosystem
  • A graphical query builder

These assets empower us to provide unparalleled stability while affording precise control over functionality and performance. The result? A seamlessly integrated ecosystem renowned for its performance, stability, and user-friendly nature.
